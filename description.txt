Цели

    1. Проходя курс по HTML / CSS накидать свою интерактивную шпаргалку по JavaScript.
    2. При получении новых знаний улучшать сайт, доводить его до нужного состояниия.
    3. Интерактивность реализовать в возможности повторять материал по минимальному количеству раз. При достижении этого количества либо исключить данную тему из списка обучения, либо запустить на повторный цикл. Сделать по аналогии с Lingua Leo.

Список классов:

Общее:
    .menu - меню
    .contents - оглавление
    .contentsJS - оглавление JS
    .contentsHTML - оглавление HTML
    .contentsCSS - оглавление CSS
    .useLinks - полезные ссылки
    .btnUp - кнопка наверх
    .list - список
    .itemsList - элементы списка
    .tascs - задачи
    
Тематические разделы:
    .jsConspectus, .htmlConspectus, .cssConspectus - основные разделы меню
    .article - тематическая карточка (статья)
        .firstTitle - заголовок
        .middleTitle - заголовок среднего уровня
    .important - важно!
    .useSoon - применить в скором времени

Предварительная планировка JavaScript

    Система обучения / напоминания
        - необходма форма, в которой можно выбрать разделы для обучения / повторения
        - при окончании повторения меняется вид раздела (стили)
        - нужен таймер, который будет выдавать/отображать количество часов проведенных за изучением/повторением материала
        - рисовать график как в поммодоро за месяц, за все время, где отображается количество затраченного времени

    Система избранного
        - выделить текст например и отправить его в избранное.
        - Скрипт сам определяет из какого раздела берется фрагмент и помещает его в соответствующий раздел в избранном
        
    Список важных ссылок
        - модальное окно добавления ссылки и названия
        - перетаскивание в границах списка. Важные моно перетеащить наверх
    Возможность подчернуть, выделить любой фрагмент текста или убрать выделение. Тип маркер
        - когда шпаргалка будет многостраничной, нужен будет список таких выделеных фрагментов. что-то похожее видел в Evernote

    Создать шаблоны для добавления новой статьи. Аргументы:
        - в какой раздел добавить
        - title
        - description
        - src to img
    Логика формы добавления новых данных в конспект:
        - выбрать раздел (HTML, CSS, JavaScript, Other)
        - заголовок темы конспектирования
        - поле ввода текста в форме должно предусматривать возможность форматировать внешний вид текста, который я помечаю как код
        - при нажатии добавить конспект попадает в нужный раздел и занимает последнее место в списке тем (то есть сортируем по дате добавления)
    
    Логика регистрации
        - Логика запроса пароля и блокировки возможностей страницы в случае ошибки
        - Вывод ошибки, запрос повторной авторизации или выхода из формы
        - При вводе данных проверка на наличие текста, на все правильно заполненные поля
